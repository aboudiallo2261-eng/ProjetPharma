DÉCISIONS ARCHITECTURALES
--------------------------
Bouton "Exporter" : SplitMenuButton avec .btn-outline
→ Raison : Le SplitMenuButton est un composant "neutre" (ni primaire ni danger).
  L'outline gris est le choix correct car il ne compete pas visuellement
  avec le bouton "Filtrer" (vert). La hiérarchie visuelle est respectée :
  Filtrer (vert, action principale) > Exporter (outline, action secondaire).

Aucun emoji conservé dans tout l'onglet.
Aucune icône SVG ajoutée (demande explicite : "pas beaucoup d'icone").
Couleur valeur stock : #27ae60 → #10B981 (unification palette Design System).

CONVENTION EMOJIS (rappel définitif)
-------------------------------------
Les emojis sont interdits dans les composants JavaFX de cette application car :
1. Rendu inconsistant selon l'OS et la version JVM
2. Signature "IA générative" immédiatement reconnaissable
3. Non compatible avec une interface professionnelle de gestion
Seul le texte propre est autorisé dans les boutons et les champs.
